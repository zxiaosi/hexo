Hexo源码构造

发表于 2021-05-15 分类于 Hexo博客 阅读次数： 评论数：
本文字数： 1.3k 阅读时长 ≈ 1 分钟

一、起因

1.1 说明

  • 本地 hexo clean && hexo g && hexo d 后，仓库的代码是已经渲染过的文件

  • 如果某天电脑突然打不开，克隆下来的也是渲染的文件，无法直接使用

1.2 计划

  • 本来打算弄两个仓库，后来觉得太麻烦，索性在一个仓库弄两个分支
  • 站点根目录下的文件都要上传的话，避免不了私密信息的泄露，所以上传的时候忽略一些文件，然后将这些文件放到其他仓库，比如Github、Gitee私人仓库，或者一些云端的网盘等。主要忽略的文件如下：
    • 站点配置文件：_conifg.yml
    • 主题配置文件：source/_data/next.yml
    • 文章：source/_posts/
