Vercel托管Github Pages

一、Vercel

1.1 Vercel简介（以前叫 ZEIT）

  • Vercel 是一个站点托管服务平台，相较于其他的站点托管服务平台CloudFlare、Netlify，主要优点体现在下面几点：

    • 国内访问速度快（首要原因）
    • 项目自动更新部署（懒人专属）
    • 可以部署多个网站或者服务 （简直不要太爽！！！）
    • 网站自动配置https，不用自己申请证书
    • 子域名的个数没有限制

  • 最重要的是Vercel个人完全免费。Vercel从开始的一个月免费20G，到后来的一个月免费100G，直到现在的个人完全免费，可见其诚意。

1.2 Vercel工作原理

  • Vercel属于复制一份Github Pages文件，然后在它给你的空间新开一个页面，同时提供cdn加速。
  • 部署比较简单，用GitHub登陆授权后，修改DNS配置域名即可。

二、Vercel使用

2.1 Vercel官网

  • 进入官网，注册账号

  • 使用GitHub或者GitLab登录

  • 选择 Github Pages 导入，或者新建一个模板（Vercel提供了诸多模板）

  • 选择作用范围，用于个人还是用于小组

  • 选择路径（一般都是在仓库根目录下 root ）

  • 选择网站对应的模板（如果网站使用了不属于原模板框架的文件，建议选择Other，部署效果和选择模板是一样的）

  • 部署成功，点击 Visit 即可看到自己的网站

2.2 绑定域名

  • 进入项目

  • 参观域名

  • 输入域名，点击 Add（Vercel会为每一个项目自带一个域名，可以点击 Edit --> Remove 删除）

三、域名配置

3.1 DNS配置

  • Add之后，它会弹出来一些需要做的配置，接下来需要去我们的域名提供商那里根据Vercel给出的要求进行域名配置。（我是在阿里买的域名 阿里域名服务）

  • 进入域名服务，点击 管理

  • 修改DNS（DNS修改之后，大概需要等10分钟才会生效）

3.2 域名解析

  • 域名解析，点击 解析

  • 添加这条解析记录

  • 如果绑定的是主域名的话，这个时候就算配置成功了！！！

3.3 CNAME配置（子域名）

  • 记住DNS解析记录

  • 回到Vercel，点击 Domains

  • 点击 View DNS Records & More

  • 填入子域名的名称，选择 CNAME，值为 cname.vercel-dns.com , TTL为 60

  • 回到项目中，等待几分钟后，即可看到绑定子域名成功

  • 接下来就可以通过我们设置的域名访问我们的网站啦！！！

四、相关事项

  • 【注意】网站根目录下需要添加CNAME文件
  • 需要更新网站时，只需要将新内容提交到Github上，Vercel就会自动部署了！
