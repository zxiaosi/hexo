Vercel托管多个网站

发表于 2021-04-07 更新于 2021-04-10 分类于 Github Pages 阅读次数： 评论数：
本文字数： 1k 阅读时长 ≈ 1 分钟

一、须知

1.1 Github Pages

  • GitHub 个人只能拥有一个 GitHub Pages。
  • yourname.github.io 只能作用于一个仓库。

1.2 Vercel

  • 参考大佬的 如何拥有多个GitHub Pages 没有成功，故选择了 Vercel 。
  • Vercel 可以复制一份Github Pages文件，然后在它的空间新开一个页面。
  • 相当于把网站托管到了 Vercel，而不是托管到 Github。

二、搭建

2.1 Github配置

  • 新建 Github 仓库，名字随意，添加 README 文件

  • 可以将自己写好的网站上传到这个仓库，比如Hexo、Docsify。或者新建一个 index.html 文件。

  • 添加 CNAME 文件，文件里写上域名

  • 仓库目录

2.2 Github Pages配置

  • 找到 Settings

  • 注意：Github Pages 不要开！不要开！不要开！！！

  • 如果你已经有仓库开启了 Github Pages，那么请关闭。Github 只允许个人拥有一个 GitHub Pages 。

  • 否则仓库会报下面的错，并且每次部署都会给邮箱发消息（Page build warning）。

  • 不如直接托管到 Vercel，此时的 Github Pages 就只是一个仓库。（CNAME文件一定要有）

2.3 Vercel 配置

  • Vercle 简单使用

  • 新建项目 New Project

  • 导入项目 import

  • 选择个人 Select

  • 继续 Continue

  • 部署 Deploy

  • 部署成功

2.4 域名配置

  • 进入项目

  • 查看域名 View Domains

  • 添加域名 Add

  • 域名添加成功

  • Vercel 默认为项目创建一个域名，可以通过 Edit --> Remove 删除。

2.5 常见问题

  • 见Vercel 托管 Github Pages
