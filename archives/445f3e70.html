一、CDN

1.1 CND 简介

  • CDN的全称是Content Delivery Network，即内容分发网络
  • 当域名访问资源时，通过DNS查找离用户最近的CDN节点（边缘服务器）的IP，并通过IP访问资源
  • 更多详情

1.2 CDN 优势

  • 提高了站点的访问速度和稳定性
  • 保证不同网络中的用户都能得到良好的访问质量
  • 访问不占用宽带，对目标网站的服务器压力小

1.3 为什么选择 jsDeliver

  • 国内也有很多CDN厂商，比如网宿科技、阿里云、百度云等，不过作为白嫖党，我选择用 jsDeliver
  • jsDeliver 在国内有很多节点，访问速度快

三、注意事项

3.1 单个文件大小

  • 上传的单个文件大小不能超过25M，否则会加载失败

3.2 最好不要用来当图床

  • Github 官方限制每个仓库不能超过 1G，且仓库超过 1G 后会有人工审核仓库内容，如果发现用来做图床，轻则删库，重则封号
  • 建议使用 PicGo+Typora+Gitee图床

3.3 最好不要放网站

  • 发布版本后，文件里会添加版本标签，网站不识别而造成一系列的错误
  • 已知：Hexo框架的网站不识别
